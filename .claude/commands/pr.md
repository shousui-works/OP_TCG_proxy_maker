# PRを作成

現在の変更に対して適当なブランチを切り、mainブランチに対してPRを作成してください。

## 手順

1. 現在の変更内容を確認する（`git status`, `git diff`）
2. 変更内容に基づいて適切なブランチ名を決める（例: `feature/add-xxx`, `fix/xxx-bug`）
3. 新しいブランチを作成してチェックアウトする
4. 変更をコミットする（適切なコミットメッセージを付ける）
5. リモートにプッシュする
6. `gh pr create` でmainブランチに対してPRを作成する

## 注意事項

- ブランチ名は変更内容を簡潔に表すものにする
- コミットメッセージは変更内容を適切に説明する
- PRのタイトルと説明も変更内容に基づいて適切に設定する
